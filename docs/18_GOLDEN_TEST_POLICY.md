# 18. Golden Test Policy

## 目的

決定論リファレンスの期待値を、実装都合で安易に書き換えないための更新規約です。ゴールデンは「現在の出力」ではなく、レビュー済み仕様を固定した回帰契約として扱います。

## 対象

- `src/battle/determinism-reference.test.ts` のv1参照戦闘
- `src/demo/reference-headless-battle-v2.test.ts` のv2参照戦闘
- 今後追加される固定イベント列、固定最終状態、固定JSON出力

## 更新できる条件

ゴールデン更新は、次のすべてを満たすPRでのみ許可します。

1. 仕様変更がPR本文に明記されている
2. 変更理由と影響する処理順・数値・イベントが説明されている
3. 実装差分とゴールデン差分が同じPRに含まれている
4. 期待されるイベント列または最終状態の変更をレビューできる
5. `pnpm test`、`pnpm demo:headless`、`pnpm lint`、`pnpm build` が成功する

不具合修正であっても、仕様上の期待結果が変わらない場合はゴールデンを変更せず、実装を修正します。

## 禁止事項

- CIを通すためだけに現在の出力を期待値へコピーする
- 差分理由を書かずにイベント数、時刻、HP、対象IDを更新する
- 100回再実行テストを削除・短縮して不安定性を隠す
- 仕様変更と無関係なゴールデン更新を同じPRへ混在させる
- 乱数、現在時刻、配列投入順を固定値の代わりに使用する

## レビュー時の確認順

1. 変更された正式仕様
2. 状態遷移とイベント処理順
3. ゴールデンの意味のある差分
4. 100回再実行の一致
5. ヘッドレス出力の手動確認

## v2参照戦闘の固定範囲

T024時点のv2は、次を一つの決定論シナリオで固定します。

- 毒の付与・行動開始ダメージ・撃破
- 火傷の付与・行動開始ダメージ・付随Attack低下
- 障壁によるHP前吸収
- 予兆の対象マス固定
- 予兆対象マスへの召喚と領域侵入
- 解決時点の占有者への予兆着弾
- タイムアウト時の残存数、残存HP割合、与ダメージ、完全同値の比較

## 手動確認

```bash
pnpm demo:headless
```

出力の `v2.summary`、`v2.timeoutResolution`、`v2.trace`、`v2.events` を確認します。ゴールデン変更時は、PR本文に旧結果から新結果へ変わる理由を記載します。
