# 12. UI / UX

## 戦闘画面

PC・タブレット横を基準とします。

- 中央: 6×3盤面
- 左: 選択ユニット詳細
- 右または上: タイムライン
- 下: ログ・手動コマンド
- 上: AUTO・一時停止・速度・ステージ情報

スマートフォンではパネルを展開式にします。

## 盤面の常時情報

- モンスター画像
- HP
- 障壁
- 主要状態最大4個
- 属性アイコン
- 次回行動簡易表示
- 予兆対象

レア度・全ステータス・全技を常時重ねません。

## タイムライン

未来10～12件:

- ユニット行動
- 予兆着弾
- 召喚
- 設置物
- フェーズ移行

技ホバー・選択時に使用後の次回位置を半透明で示します。

## 手動操作

- 固有技
- 汎用技
- 開花技
- 移動
- 待機

確定前プレビュー:

- 正確なダメージ・回復
- 撃破
- 遮蔽
- AoE
- 状態
- 強制移動
- 次回行動順位

決定論的な技は幅ではなく確定値を表示します。

## オート操作

- AUTO ON/OFF
- 一時停止
- 次の味方だけ手動
- ×1 / ×2 / ×4
- クリア済みで×8・即時シミュレーション候補

## 予兆

色だけに依存せず、枠線・模様・アイコンを併用します。

表示:

- 発動までの時間
- 属性
- 予測ダメージ
- 強制移動
- 中断条件

## 行動理由

通常ログは一行、詳細モードではスコア内訳を表示可能にします。

## 戦闘終了

- 報酬
- 修復費
- 解析度変化
- 開花・研究通知
- 個体戦績
- 再戦 / 編成 / 研究 / 次ステージ

## コレクション・編成

編成・技習得・図鑑を同じPlayerDataから操作します。

編成画面:

- 所持個体一覧と味方3×3配置
- 個体選択後にマスを押して配置・移動
- 汎用技1枠・開花技1枠
- 個体作戦・チーム作戦・技ポリシー
- `tiePriority`の並べ替え
- 編成プリセット切替・複製保存
- 選択中の編成で戦闘開始

ドラッグ&ドロップを必須にせず、タップ・クリックのみでも完結させます。

## 汎用技習得

- 習得対象個体を先に選ぶ
- 習得費と残高を明示する
- 習得済み・残高不足をボタン状態とテキストで識別する
- 習得後はその個体の編成ロードアウト候補へ即時反映する

正式な基本通貨は経済実装と接続します。接続前も習得処理はUIから分離したトランザクションとして扱います。

## 図鑑

- 種一覧カード
- 開示段階・解析度・設計図状態
- 種詳細の属性・レア度・能力値・技・統計
- 開示段階0～5に応じた段階表示
- 図鑑・編成カード・種詳細に画像用の固定領域

画像領域はアセット未設定時にプレースホルダーを表示し、文字情報やカード寸法を崩さず後から差し替えられる構造にします。

## 研究画面

T037ではコレクション画面から独立した研究画面へ移動できます。

上部:

- 現在の研究施設段階
- 現段階の最大研究可能レア度
- 次段階の基本通貨・研究データ費
- 施設更新ボタン
- 編成・図鑑へ戻るボタン

研究網:

- 開示段階1以上のノード一覧
- 状態、開示段階、隣接数
- 段階1は対象種名を伏せた`?`
- 選択状態は`aria-pressed`、枠線、模様を併用
- 段階0はDOMへ表示しない

ノード詳細:

- 開示段階トラック
- 開示済み対象種、属性、レア度、隣接情報
- 施設上限による操作ロック理由
- 候補ノードの試験研究フォーム
- 既知ノードの確定研究・標本選択フォーム
- 完了ノードの設計図解放表示

試験研究では候補方式をselect、素材種・触媒をcheckboxで選択します。最小・最大要素数、研究データ費、結果の適合数をテキストで表示します。

確定研究では研究データ、触媒、標本要件を表示し、locked個体と編成中個体を標本候補から除外します。研究操作の成否・消費ルールはUIへ実装せず、progressionの純粋トランザクションを呼び出します。

PCでは研究網と詳細の2カラム、狭い画面では1カラムへ変形します。施設更新と研究操作はタッチ・クリックのみで完結し、ドラッグ操作を要求しません。

## アクセシビリティ

- タッチだけで完結
- 色以外の識別
- `prefers-reduced-motion`
- 演出: 標準 / 簡略 / 最小
- キーボードは補助
- `?`ノードは視覚表現だけでなくテキストでも未解析と示す
- 施設ロック、残高不足、試験結果は`role="status"`または状態テキストで通知する
