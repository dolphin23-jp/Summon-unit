# 12. UI / UX

## 地域・ステージ画面

T039から起動直後の主要画面を地域・ステージ選択とします。T040では使用中セーブスロットの検証・復元後に表示します。

上部:

- 基本通貨、研究データ、触媒種数
- 現在の出撃編成
- 編成・図鑑、研究網・施設、セーブスロットへの導線
- 進行通知

地域一覧:

- 地域名、概要、表示順
- 地域解放状態
- 表示中ステージのクリア数
- 地域ポイント

ステージカード:

- 通常戦、強敵戦、地域ボス、隠し戦闘
- `OPEN / LOCKED / CLEAR`
- 主題・副題、敵数、確定報酬、地域ポイント
- 未解放理由
- 出撃ボタン
- クリア済みの場合は即時シミュレーション

隠しステージは未発見時に名称、主題、敵編成、報酬をDOMへ表示せず、「未発見ステージがある」という件数だけを示します。状態は色だけでなく、テキスト、枠線、破線、模様を併用します。

地域ポイント天井は必要ポイント、報酬ID、未到達・到達・受取済みを表示します。受取判定の正本はPlayerDataにあり、UIだけで受取状態を持ちません。

PCでは地域一覧とステージ詳細の2カラム、狭い画面では1カラムへ変形します。ドラッグ操作は要求しません。

## 進行通知

次の差分を通知します。

- 新地域解放
- 新ステージ解放
- 隠しステージ発見
- 研究ヒント・開示段階更新
- 新設計図解放
- 開花研究可能
- 地域ポイント報酬獲得

通知には固定IDを持たせ、同じ通知を重複表示しません。通知から地域、研究、編成・図鑑へ移動でき、個別または一括で確認済みにできます。進行の正本となるPlayerDataはT040からIndexedDBへ保存します。通知の既読状態そのものは現時点ではセッション内状態です。

## 戦闘画面

PC・タブレット横を基準とします。

- 中央: 6×3盤面
- 左: 選択ユニット詳細
- 右または上: タイムライン
- 下: ログ・手動コマンド
- 上: AUTO・一時停止・速度・ステージ情報

スマートフォンではパネルを展開式にします。

## 盤面の常時情報

- モンスター画像
- HP
- 障壁
- 主要状態最大4個
- 属性アイコン
- 次回行動簡易表示
- 予兆対象

レア度・全ステータス・全技を常時重ねません。

## タイムライン

未来10～12件:

- ユニット行動
- 予兆着弾
- 召喚
- 設置物
- フェーズ移行

技ホバー・選択時に使用後の次回位置を半透明で示します。

## 手動操作

- 固有技
- 汎用技
- 開花技
- 移動
- 待機

確定前プレビュー:

- 正確なダメージ・回復
- 撃破
- 遮蔽
- AoE
- 状態
- 強制移動
- 次回行動順位

決定論的な技は幅ではなく確定値を表示します。

## オート操作

- AUTO ON/OFF
- 一時停止
- 次の味方だけ手動
- 未クリア: ×1 / ×2 / ×4
- クリア済み: ×1 / ×2 / ×4 / ×8
- クリア済み: 即時シミュレーション

即時シミュレーションは通常と同じヘッドレス戦闘を実行します。UIで勝利や報酬を固定せず、通常の戦闘結果精算へ接続します。

## 予兆

色だけに依存せず、枠線・模様・アイコンを併用します。

表示:

- 発動までの時間
- 属性
- 予測ダメージ
- 強制移動
- 中断条件

## 行動理由

通常ログは一行、詳細モードではスコア内訳を表示可能にします。

## 戦闘終了

- 報酬
- 修復費
- 解析度変化
- 開花・研究通知
- 個体戦績
- 再戦 / 編成 / 研究 / 地域・ステージ

初回クリア直後から同じ結果画面の再戦でも×8を利用できます。戦闘精算後のPlayerData変更は使用中スロットへ自動保存します。

## コレクション・編成

編成・技習得・図鑑を同じPlayerDataから操作します。

編成画面:

- 所持個体一覧と味方3×3配置
- 個体選択後にマスを押して配置・移動
- 汎用技1枠・開花技1枠
- 個体作戦・チーム作戦・技ポリシー
- `tiePriority`の並べ替え
- 編成プリセット切替・複製保存
- 選択中の編成で選択中ステージの戦闘開始
- セーブスロットへの導線

ドラッグ&ドロップを必須にせず、タップ・クリックのみでも完結させます。

## 汎用技習得

- 習得対象個体を先に選ぶ
- 習得費と残高を明示する
- 習得済み・残高不足をボタン状態とテキストで識別する
- 習得後はその個体の編成ロードアウト候補へ即時反映する

正式な基本通貨は経済実装と接続します。習得処理はUIから分離したトランザクションとして扱います。

## 図鑑

- 種一覧カード
- 開示段階・解析度・設計図状態
- 種詳細の属性・レア度・能力値・技・統計
- 開示段階0～5に応じた段階表示
- 図鑑・編成カード・種詳細に画像用の固定領域

画像領域はアセット未設定時にプレースホルダーを表示し、文字情報やカード寸法を崩さず後から差し替えられる構造にします。

## 研究画面

コレクション画面および地域画面から独立した研究画面へ移動できます。

上部:

- 現在の研究施設段階
- 現段階の最大研究可能レア度
- 次段階の基本通貨・研究データ費
- 施設更新ボタン
- 編成・図鑑、地域・ステージ、セーブスロットへ戻る導線

研究網:

- 開示段階1以上の研究ノード一覧
- 状態、開示段階、隣接数
- 段階1は対象種名を伏せた`?`
- 選択状態は`aria-pressed`、枠線、模様を併用
- 段階0はDOMへ表示しない

ノード詳細:

- 開示段階トラック
- 開示済み対象種、属性、レア度、隣接情報
- 施設上限による操作ロック理由
- 候補ノードの試験研究フォーム
- 既知ノードの確定研究・標本選択フォーム
- 完了ノードの設計図解放表示

試験研究では候補方式をselect、素材種・触媒をcheckboxで選択します。最小・最大要素数、研究データ費、結果の適合数をテキストで表示します。

確定研究では研究データ、触媒、標本要件を表示し、locked個体と編成中個体を標本候補から除外します。研究操作の成否・消費ルールはUIへ実装せず、progressionの純粋トランザクションを呼び出します。

PCでは研究網と詳細の2カラム、狭い画面では1カラムへ変形します。施設更新と研究操作はタッチ・クリックのみで完結し、ドラッグ操作を要求しません。

## セーブスロット画面

- 固定3スロットを常時表示
- 保存日時、基本通貨、研究データ、所持個体数、クリア数、schema/content version、バックアップ数を表示
- 空きスロットへの保存、既存スロットの上書き、非使用中スロットのロード・削除
- 使用中スロットは削除不可
- 自動保存ON/OFFと使用中スロットだけをlocalStorageへ保存
- 保存・ロード中は操作ボタンを無効化
- current破損時にバックアップを使用した場合はテキストで通知
- IndexedDB未対応時は永続化不可を通知し、ゲーム自体は継続可能

PCでは3カラム、狭い画面では1カラムへ変形します。保存操作はドラッグを要求せず、タッチ・クリックだけで完結します。

## アクセシビリティ

- タッチだけで完結
- 色以外の識別
- `prefers-reduced-motion`
- 演出: 標準 / 簡略 / 最小
- キーボードは補助
- `?`ノードは視覚表現だけでなくテキストでも未解析と示す
- 地域・ステージ・研究施設のロック理由をテキスト表示する
- 残高不足、試験結果、即時シミュレーション結果、保存結果、復元結果は`role="status"`または状態テキストで通知する
