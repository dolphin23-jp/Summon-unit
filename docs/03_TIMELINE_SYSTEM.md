# 03. Timeline System

## 基本式

各ユニットは `nextActionTime` を持ちます。

```text
nextActionTime = currentTime + actionCost / speed
```

実装では浮動小数点誤差を避けるため、固定小数または整数スケールを使用します。たとえば時間を1000倍した整数で管理できます。

## 行動コスト目安

| 区分 | 暫定値 |
|---|---:|
| 非常に軽い | 60～70 |
| 軽い | 75～85 |
| 標準 | 100 |
| 重い | 130～150 |
| 大技 | 170～200 |
| 通常移動 | 約75 |

## 初回行動

暫定的に通常式を使います。

```text
firstActionTime = initialActionCost / speed
```

初動偏重が強すぎる場合のみ、モンスター別の初動コストを後から導入します。

## 同時刻の優先順位

1. 戦闘中の行動回数が少ない
2. 最終行動時刻が古い
3. 編成の `tiePriority`
4. 固定ユニットID

乱数は使いません。

## 予約イベント

タイムラインにはユニット行動以外も登録します。

- 予兆技の着弾
- 遅延回復
- 召喚物出現
- 時限設置物
- 効果終了
- ボスフェーズ移行

イベントには安定した順序キーを持たせます。

## 時間操作

- 一般的な支援は次回行動時刻の短縮・遅延を使う
- 素早さステータス直接変更は希少にする
- 最低行動間隔: `currentTime + 10`（暫定）
- 1回の遅延上限: 50（暫定）
- 累積遅延は元の行動間隔の200%まで（暫定）

## 追加行動

追加行動は `nextActionTime` の書き換えではなく独立イベントです。

- 追加行動から追加行動を原則発生させない
- 使用可能技を制限可能
- 通常の次回行動予定は維持可能

## UI要件

- 未来10～12件を表示
- 同一ユニットが複数回登場してよい
- 技選択中に使用後の次回位置を仮表示
- 予約イベントも同じ列に表示
