# 15. Balance Guidelines

## 仮基準個体

```text
HP 120
Attack 30
Defense 20
Speed 3
```

## ダメージ式

```text
baseDamage = attack * skillMultiplier * 100 / (100 + defense)
```

属性・与被ダメ補正後、最後に丸め、最小1を基本とします。

標準攻撃1.0 / コスト100で、標準個体へ約25ダメージ、約5回で撃破するテンポを初期仮説とします。

## 指標

```text
effectiveHP = HP * (100 + Defense) / 100
attackEfficiency = Attack * Multiplier * Speed / ActionCost
```

これに範囲、遮蔽、回復、位置、時間操作等の特殊価値を加えます。

## 回復

通常回復は受けるダメージよりやや非効率にし、永久戦闘を防ぎます。

## バフ目安

- 軽: 10%
- 標準: 20%
- 強: 30%

通常上限候補:

- Attack 50～200%
- Defense 25～250%
- Speed 50～150%
- Action Cost 50～200%
- Healing 0～250%

## レア度

高レアは明確に強くします。ただし、ステータス倍率と技倍率を同時に過剰化しないよう総合シミュレーションします。

初期垂直スライスはR1～R5のみ。R6以降の性能予算は実測後に確定します。

## 高速への制約

高速個体には次の複数を与えます。

- 低HP・防御
- 低一撃
- 狭射程
- 重い大技
- 限定条件

低速個体は行動外価値（遮蔽・反撃・常時支援）を持たせます。

## 解析・経済テンポ

- 低レア開花: 入手後2～3戦
- 3～5戦ごとに新設計図の目安
- 新設計図解放後、最低1体は早期召喚可能
- 2体目以降と別研究の資源判断を作る

## シミュレーション項目

- 勝率
- 仮想戦闘時間
- 行動回数
- 技選択率
- 移動率
- 過剰回復率
- オーバーキル
- 各個体の与ダメ・回復・肩代わり
- 速度別の総行動価値
- タイムアウト率

## T046決定論ベンチマーク

`pnpm benchmark:balance`は、導入・地域A・地域Bごとに3種類、合計9種類の固定編成を用意し、各ステージへ同じ難度帯の3編成を投入します。17ステージ×3編成の51戦を乱数なしで実行し、JSONを標準出力へ返します。

収集値:

- ステージ別・全体の勝率
- 仮想戦闘時間と行動回数
- 技行動率と技別選択率
- 移動率
- 計算ダメージに対するオーバーキル率
- 種ごとの与ダメージ、肩代わり、障壁吸収、行動価値
- 速度別の総行動価値
- 行動上限タイムアウト率

現在のヘッドレス実行系は`SINGLE_ENEMY`技だけを解決するため、回復量と過剰回復率は測定不能です。レポートでは`healingResolutionAvailable: false`と`overhealRateBasisPoints: null`を返し、0回復と混同しません。
