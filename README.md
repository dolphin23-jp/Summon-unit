# Monster Research Tactics

> Release candidate: `0.1.0-rc.1`。v0.1合格判定はNO-GOです。詳細は`docs/19_V0_1_RELEASE_READINESS.md`を参照してください。

ブラウザで動作する、モンスター収集・研究・編成型タクティクスゲームです。3×3陣形と決定論的タイムライン戦闘を採用し、オート戦闘を基本として必要な一手だけ手動介入できます。

## 開発方針

- GitHub上で仕様・タスク・実装を一元管理する
- Vercelでブラウザ版を公開する
- 一度に一つのタスクを実装する
- 戦闘ロジック、UI、コンテンツデータ、セーブを分離する
- 同じ入力から同じ結果を再現できる決定論を優先する

## 技術基盤

Vite、React 18、TypeScript strict、pnpm 10.13.1、Vitest、ESLint、Prettier、GitHub Actions、Vercelを使用します。

## ローカル開発

```bash
corepack enable
corepack prepare pnpm@10.13.1 --activate
pnpm install --no-frozen-lockfile

```

検証:


## 現在利用できる画面

起動直後は使用中セーブスロットをIndexedDBから検証・復元した後、ホーム画面を表示します。ホームから地域・ステージ、編成・図鑑、研究、召喚、セーブへ移動できます。使用中スロットに中断戦闘があれば、保存済みstable snapshotから戦闘画面を直接再開します。通常戦、強敵戦、地域ボス、隠し戦闘を解放条件つきで表示し、現在の編成、確定報酬、地域ポイント、到達報酬を確認して出撃できます。隠し戦闘は解放前に名称や敵編成を表示しません。

クリア済みステージではAUTO速度×8と即時シミュレーションを利用できます。即時シミュレーションは固定勝利ではなく、通常と同じヘッドレス戦闘を実行して勝敗・報酬を精算します。

進行通知は新地域、新ステージ、隠し戦闘、研究ヒント、新設計図、開花研究可能、地域ポイント報酬を表示し、地域、研究、編成・図鑑へ移動できます。

コレクション画面では所持個体を選び、3×3配置、汎用・開花技、個体作戦、チーム作戦、技ポリシー、行動優先度を編集し、編成プリセットを複製保存できます。編集した編成は位置・装備・AI設定を維持したまま選択中ステージの戦闘へ引き継がれます。

戦闘画面ではAUTO、一時停止、1行動進行、速度変更、次の味方だけ手動、確定前プレビュー、未来タイムライン、AI判断理由を利用できます。各stable地点で戦闘状態を保存し、手動入力選択中に終了した場合は直前stableへ戻して再開します。ステージ指定戦闘の終了時は、確定・追加・地域ポイント報酬、修復費目安、解析度変化、研究ヒント通知、個体戦績を表示し、PlayerDataへ一度だけ精算します。結果画面から再戦、編成、研究、地域・ステージ画面へ移動できます。

図鑑は開示段階0～5に対応し、カードと種詳細では`public/monsters/<speciesId>.png`の画像を表示します。画像が存在しない種では従来の頭文字プレースホルダーへ自動的に戻ります。

研究画面では施設段階、研究可能レア度、次段階費用、研究網、`?`ノード、開示済みノード詳細を表示します。候補ノードから試験研究を行い、既知ノードでは標本個体を選択して確定研究と設計図解放を実行できます。研究施設を更新すると、同じ画面上で高レア研究の操作ロックが解除されます。

セーブ画面では固定3スロットの保存、上書き、ロード、非使用中スロット削除、自動保存ON/OFF、使用中スロットのJSONエクスポート／インポートを操作できます。各スロットは保存日時、基本通貨、研究データ、所持個体数、クリア数、schema/content version、バックアップ数、中断戦闘の有無を表示します。

## セーブと永続化

PlayerData全体と任意の中断戦闘はIndexedDBへ保存します。localStorageへ保存するのは使用中スロットと自動保存設定だけです。IndexedDBが利用できない環境ではゲームを継続できますが、永続化不可を通知します。

各スロットは現在世代1件と、新しい順のバックアップ世代最大3件を持ちます。保存は次の順で行います。

1. 新しいgenerationを仮領域へ書く
2. 再読込してPlayerData、任意の戦闘snapshot、各チェックサムを検証する
3. 単一のIndexedDB transactionでcurrent pointerを切り替える
4. 旧currentをバックアップへ移し、古い世代を削除する

検証失敗時は旧currentを維持します。ロード時にcurrentが破損している場合は、バックアップを新しい順に検証して最初の有効世代を復元します。PlayerData変更時と戦闘stable地点の自動保存要求は直列化し、短時間の連続更新でpointer切替が競合しないようにしています。

旧schemaのPlayerDataは、`1→2→3→4→5→6`のように隣接versionを一段ずつ通して現行schemaへ変換します。各段階の返却versionと最終PlayerData全体を検証してから新generationとして保存し、変換元generationを最新バックアップとして保持します。変換または保存に失敗した場合は旧pointerを動かしません。

JSON転送文書は固定format名・format version・出力時刻・payload全体の正規化チェックサムを持ちます。インポートではチェックサムを確認してからPlayerData migrationと戦闘snapshot検証を行い、使用中スロットの新generationとして原子的に保存します。

## プレイヤーデータと経済

PlayerDataは種状態、固定instanceIdの所持個体、編成、経済、研究進行、研究施設段階、ステージ進行を分離します。装備状態と個体別AI設定は編成側へ保存するため、同じ個体を編成ごとに別設定にできます。

経済状態は次を保持します。

- 基本通貨
- 研究データ
- 固定catalystIdごとの個別触媒

複数資源の差分は一括検証後に確定し、一つでも負残高になる取引は全体を拒否します。汎用技習得費は共通の基本通貨から差し引かれ、通貨減少と習得済み技追加を同時に確定します。

解析度は種単位0～10000の整数basis pointsです。増加源は味方使用、敵遭遇、技観察、解析データ投入、同種標本消費の固定5種類です。経済消費と解析度増加を一つの原子的トランザクションとして処理できます。

## 地域・ステージ・報酬・ボス

地域マスターは固定`regionId`、表示順、概要、解放条件を持ちます。ステージマスターは固定`stageId`と`regionId`、地域内順序、解放条件、主題、副題、敵配置、敵AI作戦、報酬、進行ID、任意のボス定義を持ちます。新しい地域・ステージはエンジン分岐ではなくマスターデータ追加で作成します。

解放条件は常時解放、指定ステージクリア、複数ステージ全クリア、地域ポイント到達に対応します。地域画面と戦闘生成サービスの双方で同じ判定を使うため、未解放ステージを直接開始できません。

勝利精算は確定報酬、明示ロールで選ぶ追加報酬、地域ポイント、到達報酬、解析度、クリア進行、研究ヒントを一括反映します。地域ポイント到達報酬により、追加報酬が外れても固定周回数以内に素材へ到達できます。固定戦闘IDを保存して同じ結果の二重精算を防ぎます。

ボスはHPしきい値ごとのフェーズ、フェーズ別行動技表、進入時ギミックを持ちます。しきい値到達時は同じ仮想時刻へ`PHASE_CHANGE`を予約し、通常ユニット行動より先に解決します。現在は障壁展開ギミックを実装しています。

## コンテンツ検証

現行マスターはブラウザ起動時に実行時検証します。`pnpm validate:content`は同じ検証器をCLIから実行し、種・技・研究レシピ・地域・ステージ・敵AI・報酬・ボスフェーズ間の参照破損をCIで検出します。

検証ルールは既存のTypeScript assertionとnormalizerを集約しています。外部JSONやCMSを入力元にするまでは新しいschema依存を追加せず、ルール正本の重複を避けます。

## 垂直スライスコンテンツ

T044ではR1～R5の16モンスターを独立したマスターとして追加しました。全種に一体一文票、固有特性、固有技、開花技を一つずつ持たせ、汎用技はT1×5・T2×5の計10種を定義しています。

一体一文票は表示名、主役割・副役割、推奨配置、長所、弱点、相性のよい味方、敵側対策、検証要素を保持します。技の戦術的意味は固定`effectTags`へ記録し、モンスター名や技名によるエンジン分岐は追加しません。

T045では`src/content/vertical-slice-world.ts`へ、R2～R5の研究ノード10件、ヒント文40件、確定研究費、研究施設3段階、地域A/B、全17ステージ、敵編成票、初期進行を集約しました。隠し研究は毒針蛾の1件だけです。

地域Aは行動順、遮蔽、移動、遠距離、回復、毒、押し出し、障壁を9戦で段階的に扱います。地域Bは火傷、炎上床、貫通、遅延、予兆、召喚を6戦とボス2戦で組み合わせます。溶岩甲獣は2フェーズ、災炎翼竜は3フェーズです。

T046では9種類の固定編成を難度帯ごとに3編成ずつ用い、17ステージを51戦で決定論シミュレーションします。勝率、仮想時間、行動回数、技選択率、移動率、オーバーキル、個体貢献、速度別行動価値、タイムアウトをJSON出力します。最終パスは全体45勝6敗、勝率88.24%、タイムアウト0%、翼竜ボス1勝2敗です。

初期進行はR1設計図6種を解放し、草牙ネズミ、泥殻虫、風羽虫、苔芽、火種トカゲの5個体を所持します。初期編成は草牙ネズミ、火種トカゲ、風羽虫の攻撃可能な3体で、泥殻虫と苔芽は予備として保持します。導入3戦には行動順、遮蔽、移動を短文で示す非強制の案内を持たせます。

現行戦闘ランナーは`SINGLE_ENEMY`技だけを実解決するため、回復・障壁・浄化・時間短縮などの支援技はrelease blockerです。回復量と過剰回復率は未対応として`false`／`null`を出力し、ゼロ性能とは扱いません。

T044～T046マスターはブラウザ起動時と`pnpm validate:content`で、16体、42技、10研究、40ヒント文、17ステージ、敵編成票、2ボス、ID参照、初期解放順、T046調整値を検証します。

## 研究網と研究実行

研究ノードは`hidden / hinted / candidate / known / completed`の状態と開示段階0～5を持ちます。解析度、触媒所持、地域進行、隣接研究、試験失敗をヒントへ接続し、開示段階を図鑑へ同期します。

試験研究は候補として提示された方式・素材・触媒だけを受け付けます。少量の研究データだけを消費し、方式・素材・触媒・要素数の適合を個別に返します。不正解は正規化署名として保存して同じ試行を拒否し、失敗回数に応じた追加ヒントを開示します。

確定研究は`known`状態のノードに対して実行し、研究データ、触媒、R4以上で必要な標本個体を一括消費して、研究完了と設計図永久解放を同時に確定します。locked個体と編成中個体は標本へ使用できません。

研究施設は段階ごとに研究可能な最大レア度を持ちます。垂直スライスでは段階1がR2、段階2がR4、段階3がR5までを扱い、基本通貨と研究データを消費して一段階ずつ即時更新します。現実時間待機や失敗抽選はありません。

## 召喚・還元・修復・開花

解放済み設計図から、暫定基本費100とレア度倍率`1, 2, 4, 8, 16, 32, 64, 128, 256, 512`で即時召喚できます。ガチャ、失敗、待機、同種上限、所持上限はありません。instanceIdは呼び出し側が固定値を指定します。

不要個体は召喚費の70%で還元できます。locked個体と編成中個体は還元できません。condition修復は全損時に召喚費の5%を基準とし、不足割合で按分します。戦闘終了時はHP割合に応じてconditionを低下させます。

開花研究は解析度しきい値、基本通貨、研究データを満たした場合に種単位で解放します。同種の全個体が利用でき、編成メンバーごとの開花技装備は1枠です。
