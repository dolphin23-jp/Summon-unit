# T001: リポジトリ文書構成を作る

## 目的

設計書・運用規約・タスク群をリポジトリへ配置し、実装者が迷わず参照できる状態を作る。

## 背景・参照仕様

- `AGENTS.md`
- `docs/17_IMPLEMENTATION_ROADMAP.md`
- 関連する個別仕様書

## 実装範囲

- 本パッケージの `docs/`, `tasks/`, `AGENTS.md`, `README.md` を配置する
- READMEから仕様索引・ロードマップへ導線を作る
- タスクの状態一覧 `tasks/STATUS.md` を作る

## 対象外

- ゲームロジック
- フレームワーク導入
- 数値調整

## 守るべき不変条件

- UIと戦闘ロジックを分離する
- 固定IDを使用する
- 同条件で同結果となる
- このタスクの対象外を先回りしない

## 完了条件

- [ ] 文書内リンクが有効
- [ ] 仕様の確定・暫定区分が明示
- [ ] T002以降の前提が読める
- [ ] 関連する単体テストを追加する
- [ ] `pnpm test`, `pnpm lint`, `pnpm build` が成功する

## 手動確認

1. READMEまたはPR本文に実行コマンドを記載する。
2. 期待結果と実結果を確認する。
3. 決定論に関係する場合は複数回実行する。

## データ・セーブ互換性

初期タスクでは原則セーブ形式を導入しない。永続IDへ影響する変更は文書化する。

## 完了報告

- 変更概要
- 変更ファイル
- テスト結果
- 手動確認
- 残課題

## 次のタスク

`tasks/STATUS.md` の順序に従う。
