# Task 019: 対象選択・固有特性・時間操作・待機・技使用制約

## 目的

対象選択を単体敵固定から一般化し、固有特性、次回行動時刻操作、正式な待機、クールダウン・回数制限・使用条件の決定論的な基盤を実装する。

## 実装範囲

- 対象陣営と形状を分離したtargetSelector
- 敵単体、自陣単体、自陣単体+隣接、自身、自身中心領域
- 常時能力補正、被弾時反応、状態パラメータ上書きの固有特性基盤
- 固有特性を10段階効果パイプラインの対応段階へ接続
- 次回行動の短縮・遅延、最低間隔、1回遅延上限、累積上限
- 待機コスト50/65/80と連続待機リセット
- クールダウン、自行動での進行、使用回数制限、HP条件
- ヘッドレス戦闘ランナーへの待機・技使用状態統合

## 対象外

- 複数対象への実ダメージ・回復処理
- 追加行動イベント
- 完成版AI評価
- 固有特性のコンテンツ量産
- 技選択UI

## 守るべき不変条件

- 既存SINGLE_ENEMY技データはそのまま有効
- すべての順序判定と時間操作は乱数を使わない
- 次回行動時刻は必ず現在時刻+最低間隔以降
- 1回の遅延と累積遅延は上限を超えない
- クールダウンは使用者自身の行動完了時にのみ進む
- 移動・待機でクールダウンは進む
- 技または移動で連続待機回数をリセットする
