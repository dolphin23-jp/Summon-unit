# T047 — 通し結合・堅牢化

## 目的

T044～T046で制作・調整した垂直スライスを、既存の画面、戦闘、研究、経済、セーブ基盤へ接続し、プレイヤーが一つのブラウザセッションと保存データでコアループを巡回できる状態にする。

## 実装内容

- [x] 実行時カタログをT039デモからT046調整済み垂直スライスへ切り替える
- [x] ホーム画面を追加し、地域、編成、研究、召喚、セーブへの入口を集約する
- [x] ホーム→編成→出撃→結果→研究→召喚の画面循環を接続する
- [x] 解放済み設計図から決定論的instanceIdで個体を召喚する画面を追加する
- [x] 導入3戦のT046ガイダンスを地域画面へ接続する
- [x] IndexedDB保存、バックアップ復旧、JSON転送、中断戦闘再開を新カタログで継続利用する
- [x] 実行時エラーと未処理Promise rejectionを画面通知へ集約する
- [x] セーブ初期化失敗時も初期進行で継続できる縮退経路を用意する
- [x] 起動時コンテンツ検証をT047実行時カタログへ切り替える
- [x] 初期編成、初戦生成、決定論、召喚取引の統合回帰テストを追加する

## 受け入れ条件

- 起動直後にT046の16種、17ステージ、10研究ノードを使う
- 初期5個体と3体編成で地域A-01へ出撃できる
- 戦闘終了後の報酬・進行が同じPlayerDataへ精算される
- 研究完了後に召喚画面へ移動し、設計図から個体を追加できる
- 保存、ロード、バックアップ復旧、JSON転送、中断戦闘再開が同じカタログで検証される
- 同じ初期データから生成する初戦定義と召喚IDが決定論的である
- 既存CIのcontent validation、test、headless demo、AI benchmark、balance benchmark、lint、buildが通る

## 非対象

- v0.1合格条件の最終判定、Vercel本番確認、タグ付け（T048）
- 新モンスター、新地域、新戦闘効果の追加
- クラウド同期、PvP、R6以降
