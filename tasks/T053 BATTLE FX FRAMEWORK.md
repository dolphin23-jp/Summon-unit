# Task 053: 戦闘演出フレームワーク（フローティングダメージ・モーション・技エフェクト拡張点）

## 目的

戦闘イベントを視覚演出へ変換するプレゼンテーション層を新設し、ダメージ数値ポップ・被弾/回復表現・移動トゥイーン・行動強調を実装する。将来、技ごとの固有エフェクトをデータ登録だけで追加できる拡張点を確定する。

## 背景・参照仕様

- `docs/03_TIMELINE_SYSTEM.md`、`docs/07_BATTLE_EFFECTS.md`、`docs/12_UI_UX.md`（演出3段階）、`docs/16_DATA_SCHEMAS.md`（イベント購読モデル）
- 前提タスク: T051（スプライト盤面）、T052（レイアウト・トークン）
- 設計方針: 戦闘カーネルは一切変更しない。既にUIが購読しているイベントログ（ダメージ適用・状態付与・移動・召喚・撃破・予兆解決など）を「演出キュー」へ写像する `fx-mapper`（純関数）＋再生コンポーネントを新設する。演出はUI状態のみで完結し、決定論に影響しない

## 実装範囲

- `src/ui/fx/` 新設:
  - `fx-mapper.ts`: BattleEvent列 → FxCommand列（種別・対象マス・数値・色・持続ms）への純関数変換。単体テスト可能にする
  - `FxLayer.tsx`: 盤面上へ絶対配置で再生するオーバーレイ（CSS transition/keyframes基盤。外部ライブラリ禁止）
- v1で実装する演出:
  - フローティング数値: ダメージ（属性色）、回復（緑）、障壁吸収（青系・「吸収」）、軽減/無効、毒・火傷の継続ダメージ（小さめ）
  - 被弾フラッシュ、撃破フェードアウト、蘇生フェードイン
  - 移動・強制移動のマス間トゥイーン（押し出し失敗時は揺れで表現）
  - 行動中ユニットの強調リング、予兆着弾マスの点滅（着弾1行動前から）
  - 技使用時の汎用エフェクト: 使用者の発光＋対象へ属性色のヒットマーク
- 技固有エフェクトの拡張点:
  - `skillFx` マスター（`skillId` → プリセット名＋色/形状/持続のパラメータ）。未登録技は属性ベースの汎用にフォールバック
  - プリセットは最小3種（斬撃線・投射軌道・範囲波紋）を実装し、追加はデータのみで可能なことをテストで保証
- 演出制御:
  - 既存の演出3段階と連動: 標準=全演出 / 簡略=数値ポップと強調のみ / 最小=演出なし（即時反映）
  - `prefers-reduced-motion` は自動的に「最小」相当へ
  - ×8速度・即時シミュレーションでは演出をスキップし、結果整合を保つ
  - 演出中も入力は非ブロッキング（一時停止・手動割込を阻害しない）

## 対象外

- サウンド、パーティクルライブラリ等の新規依存、スプライトシートアニメ（画像は現行1枚のまま）
- カットイン・カメラ演出

## 変更可能な領域

- `src/ui/**`（新設 `src/ui/fx/` を含む）、`skillFx` マスター（`src/content/`）、`docs/07`/`docs/12` の演出節

## 守るべき不変条件

- `src/battle/**` 無変更。イベントログのスキーマ変更なし
- 決定論・stable snapshot・戦闘再開のテストゴールデン不変（演出はUI層のみ）
- 演出のスキップ/中断（画面離脱・リセット）でUI状態が壊れないこと（FxLayerは常に破棄安全）
- 最小演出時、従来と同等の反応速度を維持
