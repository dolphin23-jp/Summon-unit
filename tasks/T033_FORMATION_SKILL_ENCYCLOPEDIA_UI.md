# T033: 編成UI・技習得・図鑑

## 目的

T032のPlayerDataと編成モデルをブラウザUIへ接続し、所持個体から3×3編成を作成し、技装備・AI設定を変更し、その編成で対話型戦闘を開始できる状態にする。個体ごとの汎用技習得と、開示段階に応じた図鑑・種詳細の表示土台も同じ画面へ統合する。

## 実装内容

### 編成編集UI

- 所持個体カードから対象を選び、味方3×3盤面の任意マスへ配置・移動
- 占有マスへ配置した場合は既存個体と交代
- 編成プリセットの切替と、現在編成を固定IDの新規プリセットとして複製保存
- 編成メンバーごとの汎用技1枠・開花技1枠
- 個体作戦8種、チーム作戦4種、装備中技のポリシー5種
- `tiePriority`の前後入替
- 編成から個体を外す操作
- 空編成では戦闘開始を禁止

### 編成からの戦闘開始

- `createFormationBattleDefinition`でPlayerDataの編成を対話型戦闘定義へ変換
- battleUnitIdは`ally:<instanceId>`の固定形式
- 3×3位置、装備済み汎用・開花技、`tiePriority`、個体別AI設定を戦闘へ引き継ぐ
- 敵側は既存の標準デモ編成を使用
- 戦闘結果画面の「編成」および戦闘中の「編成へ戻る」からコレクション画面へ復帰
- `MinimalBattleScreen`は既定の標準戦闘を維持しつつ、外部定義を受け取れる

### 汎用技習得

- `GenericSkillLearningState`で個体別のT033用訓練通貨残高を保持
- 習得対象個体、汎用技、習得費を指定する純粋トランザクション
- 残高不足、重複習得、非GENERIC技、不明IDを拒否
- 成功時は選択個体の習得済み技と残高を同時に更新し、元入力を変更しない
- 習得済み技はPlayerDataへ保存され、編成ロードアウト候補へ即時反映
- 正式な基本通貨・経済トランザクションはT034で接続するため、訓練通貨台帳は交換可能な境界として分離

### 図鑑・種詳細

- 種一覧カード、解析度、開示段階、設計図状態、所持数
- 開示段階0～5を表示側で解釈し、未開示・属性/レア度・能力値・技・統計を段階的に表示
- T035の研究網・ヒントトリガ接続前は、PlayerSpeciesStateの`encyclopediaStageId`を表示入力として使用
- 図鑑カード・所持個体カード・種詳細に将来の画像差し替え用アイコン領域を常設
- 現時点では固定文字のプレースホルダーで、画像アセットIDはマスターへ追加しない

### UI構成

- 編成、技習得、図鑑の3タブ
- PCでは2カラム、スマートフォンでは1カラムへ変形
- タッチで全操作を完結
- 選択状態は色だけでなく枠線・模様・`aria-pressed`を併用
- `prefers-reduced-motion`ではタブ・ボタンのトランジションを停止

## 不変条件

- すべての編集結果をT032の`createPlayerData`で再検証する
- 一つの編成内で個体・位置・`tiePriority`を重複させない
- 装備汎用技は個体が習得済み、装備開花技は種で解放済み
- ロードアウトから外した技を参照するAIポリシーを同じ更新内で除去する
- 習得費と残高は非負の安全な整数
- 現在時刻・乱数・表示名を永続ID生成や順序決定に使用しない
- 編成プリセットIDは既存数から導出し、重複時は固定的に次番号へ進める

## テスト

`src/progression/t033-collection.test.ts`で以下を検証する。

- 個体別通貨の消費と汎用技習得
- 他個体残高・入力PlayerDataの非破壊
- 残高不足、重複習得、非GENERIC技の拒否
- 所持個体の配置と位置保存
- `tiePriority`の入替
- 編成プリセットの独立複製
- 技を外した際の無効AIポリシー除去
- 編成から生成した戦闘定義の個体ID、位置、装備技、AI設定、敵編成

検証コマンド:

```bash
pnpm test
pnpm demo:headless
pnpm benchmark:ai
pnpm lint
pnpm build
```

## 手動確認

1. `pnpm dev`で編成タブを開く。
2. 所持個体を選び、3×3盤面で配置を変更する。
3. 汎用・開花技、個体作戦、チーム作戦、技ポリシーを変更する。
4. プリセット保存で別編成が作成・選択されることを確認する。
5. 技習得タブで個体を選び、通貨減少と習得済み表示を確認する。
6. 図鑑タブで開示段階ごとの情報差と画像プレースホルダーを確認する。
7. 「この編成で戦闘開始」を押し、編集した位置・装備・AI設定で戦闘が開始されることを確認する。
8. 戦闘画面または結果画面から編成へ戻る。
