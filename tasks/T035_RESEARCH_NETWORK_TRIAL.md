# T035: 研究網・情報開示・試験研究

## 目的

研究レシピを一本道ではなく固定IDのネットワークとして表現し、プレイヤーごとの研究状態を`hidden / hinted / candidate / known / completed`で保存する。解析度、触媒入手、地域進行、隣接研究、試験失敗を情報開示へ接続し、候補範囲内だけで安全に試験研究を実行できる境界を作る。

## 実装内容

### 研究網マスター

- 固定`nodeId`、対象`speciesId`、隣接ノード、研究方式、候補範囲、正解レシピ、試験費、ヒントを分離して定義
- 研究方式は融合・系譜・収束・超越の固定4種類
- ノードID、ヒントID、候補ID、レシピIDの重複と空文字を拒否
- 対象種・素材種・隣接ノードの参照整合を検証
- 正解レシピが候補範囲と要素数範囲に含まれることを検証

### プレイヤー研究状態

各ノードについて次を保存する。

- `status`: `hidden / hinted / candidate / known / completed`
- `disclosureStage`: 0～5
- 開示済み`hintId`
- 正規化済みの不正解試験署名

マスターに存在するがセーブ入力にないノードは`hidden`・段階0として補完する。ノードは固定ID順へ正規化し、入力と結果を変更可能な参照として外部へ公開しない。

### 情報開示

次の固定5トリガを実装する。

1. 関係種の解析度
2. 触媒所持
3. 地域・ボス等の進行ID
4. 隣接ノードの`known`または`completed`
5. 不正解試験回数

- ヒントは開示段階1～4を持つ
- 同一評価内で隣接ノードが連鎖開示される場合も、固定ノード順と反復評価で決定論的に収束させる
- 開示段階から研究状態を導出する
- 対象種の図鑑段階を研究開示段階へ同期し、既存段階を下げない
- 完了状態はヒント評価で巻き戻さない

### 試験研究

- `candidate`状態のノードだけ実行可能
- マスターが提示する方式・素材種・触媒・要素数の候補範囲外を拒否
- 少量の研究データだけを消費
- 個体と触媒は試験では消費しない
- 方式、各素材、各触媒、素材数、触媒数を要素別に判定
- 完全一致時は`known`・開示段階4へ移行
- 不正解時は固定並びの正規化署名を保存
- 同じ不正解署名の再試行を拒否
- 不正解回数トリガを即時再評価し、追加ヒントを開示
- 候補違反、重複署名、研究データ不足では結果を返さず、部分変更を公開しない

### PlayerData統合

- `PlayerData.research`を追加
- `PlayerDataContentCatalog.researchNodes`を追加
- 既存T032～T034入力との互換性のため、入力時は研究状態と研究マスターを省略可能
- 正規化後は空状態を含むfreeze済み研究状態を必ず保持

## 不変条件

- 研究状態と開示段階は対応関係を崩さない
- 開示段階は0～5
- ノード、ヒント、素材、触媒、署名は固定ID順
- 試験研究は研究データ以外を消費しない
- 不正解署名は同一ノード内で一意
- 現在時刻、乱数、表示名を状態遷移や署名へ使用しない
- 入力PlayerData、研究状態、マスター配列を変更しない

## テスト

`src/progression/research.test.ts`で以下を検証する。

- 欠落ノードの`hidden`補完と固定ID順正規化
- 解析度・触媒・進行・隣接ヒント
- 図鑑段階への非降格同期
- 試験研究の研究データ消費と触媒・個体非消費
- 方式・素材・触媒・要素数の個別判定
- 正解時の`known`移行
- 不正解署名保存と試験ヒント開示
- 候補範囲外、重複署名、残高不足の原子的拒否
- 100回再実行のJSONバイト一致
