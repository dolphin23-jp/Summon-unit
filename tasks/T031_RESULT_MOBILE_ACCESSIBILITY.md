# T031 終了画面・モバイル・アクセシビリティ

## 目的

Phase 4の戦闘UIを、戦闘開始から終了後の導線までタッチ操作だけで完結できる状態へ仕上げる。結果表示はT038の報酬システム接続前でも構造を確認できるよう、進行状態を変更しない明示的なスタブとして実装する。

## 実装内容

### 戦闘終了画面

- 戦闘終端時に勝利・敗北・引き分け・行動上限を文字、枠線、模様で識別して表示する。
- 戦闘ログと終了時ユニット状態から、次の個体戦績を決定論的に集計する。
  - 生存・戦闘不能
  - 残存HP
  - 行動回数
  - 技使用回数
  - 与ダメージ・被ダメージ
  - 障壁吸収量
  - 撃破数
- 報酬、修復費、解析度、開花・研究通知を表示する。
- 報酬・解析度・通知はT038接続前の表示用スタブであり、プレイヤー進行データは変更しない。
- 再戦は現在の標準戦闘を再生成する。
- 編成・研究・次ステージは後続タスクの接続予定を画面内で通知し、結果状態を保持する。

### モバイル

- タイムラインとログ・行動理由をネイティブの `details` / `summary` を用いた展開式パネルにした。
- スマートフォン幅では展開ボタンを表示し、タッチだけで開閉できる。
- デスクトップでは既存の2カラム配置を維持する。
- 個体戦績表は横スクロール可能とし、フォーカス可能な領域としてキーボード利用者にも明示する。
- 終了後の主要ボタンは狭い画面で1列に並び、44px以上の操作領域を維持する。

### アクセシビリティ

- 陣営は色に加えて `A` / `E` と「味方」「相手」を表示する。
- 予兆は色に加えて `!`、枠線、斜線模様を併用する。
- 結果は `WIN` / `LOSE` / `DRAW` / `LIMIT`、枠線種別、背景模様を併用する。
- 解析通知・報酬種別にも文字記号を付け、色だけに依存しない。
- 演出レベルを「標準」「簡略」「最小」の3段階で選択できる。
- 選択した演出レベルを `localStorage` に保存する。保存不可の場合も現在セッションでは選択を維持する。
- 保存済み設定がない場合は `prefers-reduced-motion` を初期値へ反映する。
- `prefers-reduced-motion: reduce` ではCSS側でも遷移・アニメーション時間を最小化する。
- 結果、ナビゲーション通知、戦闘ログはライブリージョンで読み上げ可能にする。

## 設計

- `battle-result-view.ts` はReactから分離した純関数で、同じ戦闘結果から同じ終了画面モデルを生成する。
- 報酬スタブは結果種別に応じた固定整数であり、ランダム性を持たない。
- 修復費は味方の欠損HPと戦闘不能数から固定式で算出する。
- `battle-accessibility.ts` は演出設定の許容値と初期値解決を純関数として保持する。
- T030の手動操作パネルは `ManualActionPanel.tsx` へ分離したが、候補選択・プレビュー・明示確定の挙動は変更していない。

## テスト

`battle-result-view.test.ts` で次を固定した。

- 終了画面の個体数・個体状態・行動回数がヘッドレス結果と一致する。
- 個体戦績の被ダメージ合計がイベントログの適用ダメージ合計と一致する。
- 同じ結果から生成した終了画面モデルがbyte-identicalになる。
- 演出設定は標準・簡略・最小だけを受理する。
- 保存済み設定を優先し、未設定時はreduced-motion環境を反映する。

最終検証コマンド:

```bash
pnpm test
pnpm demo:headless
pnpm benchmark:ai
pnpm lint
pnpm build
```
