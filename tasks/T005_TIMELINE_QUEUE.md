# T005: 安定順序を持つタイムラインキュー

## 目的

ユニット行動と予約イベントを時刻順に処理できる純粋データ構造を作る。

## 背景・参照仕様

- `AGENTS.md`
- `docs/17_IMPLEMENTATION_ROADMAP.md`
- 関連する個別仕様書

## 実装範囲

- ScheduledEvent
- time/priority/sequenceの比較
- push/pop/peek
- 同時刻安定順
- キャンセル

## 対象外

- 行動コスト計算
- 技処理
- UI

## 守るべき不変条件

- UIと戦闘ロジックを分離する
- 固定IDを使用する
- 同条件で同結果となる
- このタスクの対象外を先回りしない

## 完了条件

- [ ] 同じ入力順から同じ出力
- [ ] 同時刻優先が仕様どおり
- [ ] キャンセル済みを実行しない
- [ ] 関連する単体テストを追加する
- [ ] `pnpm test`, `pnpm lint`, `pnpm build` が成功する

## 手動確認

1. READMEまたはPR本文に実行コマンドを記載する。
2. 期待結果と実結果を確認する。
3. 決定論に関係する場合は複数回実行する。
