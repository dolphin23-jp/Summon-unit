# T010: 決定論リファレンステスト

## 目的

固定編成を複数回実行し、結果とイベント列が完全一致することを保証する。

## 背景・参照仕様

- `AGENTS.md`
- `docs/17_IMPLEMENTATION_ROADMAP.md`
- 関連する個別仕様書

## 実装範囲

- テスト用2～4種
- 固定戦闘シナリオ
- イベントスナップショットまたはハッシュ
- 100回反復
- ID・順序の安定化

## 対象外

- AI
- 乱数技
- UI

## 守るべき不変条件

- UIと戦闘ロジックを分離する
- 固定IDを使用する
- 同条件で同結果となる
- このタスクの対象外を先回りしない

## 完了条件

- [ ] 100回同一結果
- [ ] テスト順序に依存しない
- [ ] 失敗時に差分が読める
- [ ] 関連する単体テストを追加する
- [ ] `pnpm test`, `pnpm lint`, `pnpm build` が成功する

## 手動確認

1. READMEまたはPR本文に実行コマンドを記載する。
2. 期待結果と実結果を確認する。
3. 決定論に関係する場合は複数回実行する。

## データ・セーブ互換性

初期タスクでは原則セーブ形式を導入しない。永続IDへ影響する変更は文書化する。

## 完了報告

- 変更概要
- 変更ファイル
- テスト結果
- 手動確認
- 残課題
