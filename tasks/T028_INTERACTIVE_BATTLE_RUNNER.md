# T028 — 対話型戦闘ランナー

## 目的

再生専用だった戦闘UIと一括実行ランナーを、決定論を保ったまま1行動ずつ停止・再開できる実行モデルへ置き換える。UIは戦闘ルールを持たず、カーネルが公開するスナップショットとイベントログを購読する。

## 完了条件

- [x] `runHeadlessBattle` の一括ループを、状態を保持する `createInteractiveBattleRunner` と1行動単位の `step` へ分解する
- [x] 既存の `runHeadlessBattle` は互換ラッパーとして維持する
- [x] 行動選択を `evaluateConfiguredAiDecision` へ切り替え、個体・チーム作戦と技ポリシーを利用する
- [x] 戦闘状態、イベントログ、仮想時刻、行動数、判断理由を不変スナップショットとして購読できる
- [x] AUTO ON/OFF、一時停止、1行動進行、×1 / ×2 / ×4をUIから操作できる
- [x] 「次の味方行動だけ手動」の割込点を設け、手動確定後はAUTOへ復帰する
- [x] 同じ定義と手動入力から同じ結果になる回帰テストを追加する

## 実装判断

- UIの速度は実時間の待機間隔だけを変え、仮想時刻・行動順・AI判断へ影響させない。
- 手動割込では、設定済みAIが評価した合法候補と正式待機だけを提示する。任意対象選択と確定前プレビューの完成はT030へ残す。
- T013の最小AIは既存単体テスト用に残すが、戦闘ランナーからは使用しない。
- AI設定未指定の個体は `STANDARD / STABLE_CLEAR` を使用する。
- `battle/` はReactやDOMへ依存せず、UIはイベントログを順番に適用して表示状態を作る。

## 対象外

- T029の状態アイコン、タイムライン、詳細理由パネル
- T030の技スロット別操作、任意対象・移動先指定、確定前プレビュー
- 予兆・召喚等の予約イベントを対話ランナーで実行する一般化
- セーブデータへの途中戦闘スナップショット永続化
