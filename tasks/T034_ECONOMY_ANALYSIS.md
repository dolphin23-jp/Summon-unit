# T034: 経済・解析度

## 目的

基本通貨・研究データ・個別触媒をPlayerDataへ統合し、複数資源を一つの原子的トランザクションとして更新できる境界を作る。種ごとの解析度を0～10000の整数basis pointsで管理し、仕様で定めた5種類の増加源を決定論的に適用できるようにする。

## 実装内容

### 経済状態

- `PlayerData.economy`へ基本通貨、研究データ、触媒残高を追加
- 触媒は固定`catalystId`と非負の安全な整数数量で保存
- 触媒残高を固定ID順へ正規化し、0残高を保存しない
- デモPlayerDataをschemaVersion 2へ更新
- コレクション画面のヘッダーへ基本通貨・研究データ・触媒総数を表示

### 原子的資源トランザクション

- 基本通貨差分
- 研究データ差分
- 複数触媒差分
- 同じ触媒IDの差分を一度集約してから確定
- すべての入力・結果を安全な整数として検証
- 一つでも負数またはオーバーフローになる場合は結果を返さない
- 入力状態は変更せず、成功時のみfreeze済みの新状態を返す

### 解析度

増加源を次の固定5種類として定義する。

1. `ALLY_USE` — 味方として使用
2. `ENEMY_ENCOUNTER` — 敵として遭遇・撃破
3. `SKILL_OBSERVATION` — 技の使用・観察
4. `DATA_INVESTMENT` — 解析データ投入
5. `SPECIMEN_CONSUMPTION` — 同種標本消費

- 増加量は正の安全な整数basis points
- 同じ種・同じ増加源の入力を集約
- 種ID順、続いて上記の増加源順で適用
- 10000を超える分は切り捨て、要求量と実適用量をreceiptへ保持
- 未登録種、未知の増加源、0以下の増加を拒否
- 100回再実行でJSONバイト一致を検証

### 経済と解析度の同時確定

`applyPlayerProgressionTransaction`で、研究データや触媒の消費と解析度増加を一つの操作として扱う。経済側または解析側のどちらかが不正ならPlayerDataを返さず、部分適用を外部へ公開しない。

### 汎用技習得の正式接続

- T033の個体別訓練通貨台帳を撤去
- 汎用技習得費を共通の基本通貨から支払う
- 通貨減少と個体の習得済み技追加を同時に確定
- 残高不足、重複習得、非GENERIC技を拒否
- UIはPlayerDataの基本通貨だけを残高の正本として参照

## 不変条件

- 基本通貨、研究データ、触媒残高は負にならない
- 数量と差分は安全な整数
- 触媒IDは空文字不可、状態内で一意
- 解析度は0～10000
- 解析増加源は固定5種類
- 一つの操作で一部の資源だけを変更しない
- 現在時刻、乱数、表示名を取引順序やIDへ使用しない
- 入力PlayerData、EconomyState、配列を変更しない

## テスト

`src/progression/economy-analysis.test.ts`で以下を検証する。

- 触媒の固定ID順正規化
- 基本通貨・研究データ・複数触媒の混合取引
- 重複触媒差分の集約
- 負数結果時の全取引拒否と入力非破壊
- 解析度5増加源
- 同種・同源の集約と固定適用順
- 10000上限と要求量・実適用量
- 経済支払いと解析度増加の同時確定
- 100回再実行のバイト一致
- 不明種の拒否

既存のT032/T033テストでは、経済状態のRepository保存と汎用技習得時の基本通貨消費も検証する。

検証コマンド:

```bash
pnpm test

pnpm benchmark:ai
pnpm lint
pnpm build
```
