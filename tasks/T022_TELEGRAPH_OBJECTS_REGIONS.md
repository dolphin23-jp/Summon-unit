# Task 022: 予兆・設置物・領域

## 目的

予兆技を決定論的な予約イベントとして扱い、宣言時に固定した対象マスへ解決時点の占有者が着弾対象となる基盤を作る。あわせて、盤面を占有して撃破・遮蔽対象となる設置物と、盤面を占有せず侵入時に発動する領域を実装する。

## 守るべき不変条件

- `telegraphDelay` 未指定または `null` の既存技は即時技として有効なままにする
- 予兆はユニットIDではなく対象マスを固定し、解決時点の占有者へ着弾する
- 同時刻では予兆解決を通常ユニット行動より先に処理する
- 対象マス、領域マス、複数領域の順序は固定座標・固定IDで安定させる
- 生存ユニットと有効な占有型設置物は同じマスを占有しない
- 撃破された設置物は占有・DIRECT遮蔽へ参加しない
- 非占有領域はユニットや設置物の占有を妨げない
- 領域内から同じ領域内への移動では再発動せず、領域外から内への侵入時だけ発動する
- 強制移動と召喚による侵入も通常移動と同じ発動規則を使う
- 同じ入力から同じ解決結果を返し、入力状態を変更しない

## データ・セーブ互換性

- `SkillDefinition.telegraphDelay` は任意項目とし、未指定・`null` は即時技とする。
- 設置物・領域は戦闘中の独立状態として追加し、永続セーブは未実装のため既存セーブ互換性への影響はない。
- 予約イベントkindは既存の `TELEGRAPH_RESOLVE` を使用し、タイムラインのスキーマを変更しない。

## 仕様上の判断

- 予兆解決優先度は `-100` とし、同一仮想時刻の通常ユニット行動より先に解決する。
- 予兆の対象マスは内部6×3盤面の固定順へ正規化する。
- 設置物のDIRECT遮蔽は、攻撃者から相手側へ向かう同列上の有効な `blocksDirect` 設置物を対象とする。
- 領域は `createdAt <= currentTime < expiresAt` の半開区間で有効とする。
- 召喚侵入は移動元を持たず、配置先が有効領域内なら侵入として扱う。
