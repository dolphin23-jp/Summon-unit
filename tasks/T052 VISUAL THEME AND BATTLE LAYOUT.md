# Task 052: ビジュアルテーマ統一・戦闘画面モバイルレイアウト再構成

## 目的

タスク別CSSの寄せ集め（t031/t033/t037/t039/t040…）を整理してデザイントークンへ集約し、「管理画面的な文字の壁」から視覚階層のあるゲーム画面へ引き上げる。戦闘画面はモバイル1画面型に再構成する。

## 背景・参照仕様

- `docs/12_UI_UX.md`（展開式パネル、色以外の識別、reduced-motion、演出3段階）
- 前提タスク: T049（名称）、T051（スプライト盤面）
- 現状の問題:
  - CSSがタスク単位のファイルに分散し、色・余白・角丸・書体が画面ごとに微妙に不一致
  - 全画面が同一ダークトーンで、地域の雰囲気（翠風/灼熱）が配色に反映されない
  - 戦闘画面が縦積み（盤面→操作→タイムライン→ログ→AI理由）でモバイルでは長大なスクロール
  - 次回行動が「+14286t」等の生の仮想時刻で直感的でない
  - 結果画面の報酬が「K 触媒」など1文字記号、ホームの数値が縦積みカードで冗長、通知が長文テキスト行

## 実装範囲

- デザイントークン集約: `src/ui/theme.css`（新設）にCSSカスタムプロパティで色・余白・角丸・影・書体スケールを定義し、既存CSSを段階的に参照へ置換。タスク別CSSは統合し重複を削除
- 地域テーマカラー: 地域マスターへ `themeColor` を追加し、地域画面・戦闘画面の背景/アクセントへ反映（翠風=緑系、灼熱=赤橙系）。コントラスト比はWCAG AAを維持
- 戦闘画面のモバイル1画面化:
  - 上部固定: 盤面（T051スプライト）＋勝敗/経過の最小ヘッダ
  - 下部: 「操作」「タイムライン」「ログ・AI理由」の3タブ切替（既存 `MobileDisclosure` を流用可）。デスクトップ幅では従来どおり並列表示
  - 手動割込・確定前プレビューはタブ状態に関わらず前面に出る
- 時刻表示の直感化: 生の仮想時刻を隠し、「次はこのユニットから3番目」「あと2行動で予兆着弾」の相対表現へ（`battle-screen-view` に表示用フィールドを追加。カーネル値は不変）。詳細モードでは従来の t 値も併記
- 報酬・通貨アイコン: 1文字記号をSVGアイコン＋ラベルへ置換（基本通貨/研究データ/触媒/地域P/設計図/修復費）
- ホーム進行サマリーを2列グリッド＋アイコン化、通知を「種別アイコン＋1行＋移動」形式へ圧縮

- スナップショット系テストのゴールデン不変（表示文言を変える場合は差分理由をPRに明記して更新）
- ビルドサイズ: アイコンはインラインSVGとし、新規ランタイム依存を追加しない
