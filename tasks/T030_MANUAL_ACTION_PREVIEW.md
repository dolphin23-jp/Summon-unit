# T030 手動操作・確定前プレビュー

## 目的

T028で追加した次の味方行動への手動割込を、実際に一手を選び、結果を確認してから確定できる操作へ完成させる。UI上の概算ではなく、戦闘カーネルと同じ計算経路を用いて確定前の結果を表示する。

## 実装内容

- 戦闘ユニット定義へ `equippedSkillIds` を追加し、固有技に加えて汎用技・開花技をロードアウトできるようにした。
- AUTO用の技ポリシーと手動候補列挙を分離した。`AUTO_DISABLED` の技も、使用可能なら手動候補には表示される。
- 手動候補へ行動カテゴリ、技ID、対象、対象位置、正確な結果プレビューを付与した。
- 手動操作を次の明示的な段階へ分けた。
  1. 固有技・汎用技・開花技・移動・待機から行動を選択
  2. 対象または移動先を指定
  3. 確定前プレビューを確認
  4. 「この行動を確定」で実行
- 候補選択・フォーカス・ホバーだけでは戦闘状態を変更しない。
- 盤面上で選択位置と影響範囲を表示する。
- タイムラインの仮次回行動は、UI側で再計算せず、ランナーが返す正確な次回時刻と移動後位置を利用する。

## プレビュー内容

`manual-action-preview.ts` は既存の戦闘解決器を再利用し、次を不変データとして返す。

- 行動コスト
- 次回行動時刻とタイムライン上の順位
- 選択対象・実対象・遮蔽で停止させたユニット
- 影響マスとAoE範囲
- 計算ダメージ
- 障壁吸収量と障壁残量
- HP変化と撃破見込み
- 肩代わり先と肩代わりユニットの被害
- 状態変化と強制移動を表示するための結果欄

現在のT030デモ技は直接ダメージ技のみであるため、状態変化と強制移動の欄は「変化なし」「なし」と表示する。状態付与・強制移動を持つ技データが戦闘ランナーへ接続された際は、同じプレビュー結果欄へ追加する。

## 決定論と実行一致

- 技候補はT025の候補列挙・結果プレビューを基礎にする。
- 障壁と肩代わりは実行時と同じ `resolveBarrierGuardDamage` を使う。
- 次回行動順位は実際の `ScheduledEvent` 比較規則を使う。
- 確定時は選択済み `AiActionEvaluation` を実行へ渡し、表示した対象・計算値と同じ解決経路を使う。
- ヘッドレス決定論基準戦闘と、手動技を装備した対話用デモ戦闘は分離し、既存の100回再実行テスト性能を維持する。

## テスト

`manual-action-preview.test.ts` で次を固定した。

- AUTO無効技を含め、固有・汎用・開花・移動・待機の5カテゴリが手動候補へ出る。
- 候補の参照だけでは戦闘状態が変化しない。
- プレビューしたHP・障壁と、確定後の戦闘状態が一致する。
- 障壁吸収と肩代わりのプレビューが確定結果と一致する。

最終検証コマンド:

```bash
pnpm test
pnpm demo:headless
pnpm benchmark:ai
pnpm lint
pnpm build
```

## 対象外

- 状態付与・強制移動を持つ新しいコンテンツ技の追加
- 戦闘終了画面
- モバイル向け展開式パネルと演出設定

これらは後続タスクで接続する。
