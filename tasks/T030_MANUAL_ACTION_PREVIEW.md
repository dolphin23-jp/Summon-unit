# T030 — 手動操作・確定前プレビュー

## 目的

T028の手動割込とT029の戦闘画面を、行動・対象を選択してから同じ戦闘解決結果を確認し、明示的に確定できる操作へ完成させる。

## 実装範囲

- [x] ユニットごとの固有技・装備汎用技・装備開花技を戦闘ランナーへ接続
- [x] AIポリシーによる除外とは独立して、全合法手動候補を列挙
- [x] 技／移動／待機を選び、技対象・移動先を二段階で指定してから確定
- [x] 盤面セルとタッチ用対象ボタンの双方から対象を選択
- [x] 障壁・肩代わり込みHP実損、撃破、DIRECT遮蔽、AoE対象マスを確定前表示
- [x] 状態付与と強制移動を実際の解決器で事前計算し、確定結果と一致させる
- [x] 行動コスト、次回行動時刻、タイムライン予測順位を表示
- [x] プレビューと確定後状態の一致を回帰テスト

## 仕様上の判断

- `HeadlessBattleUnitDefinition.equippedSkillIds` は汎用・開花技の装備スロットを表し、固有技は種マスターから自動追加する。
- AI設定の `AUTO_DISABLED` 等はAUTO選択にのみ作用し、手動時は使用条件・クールダウン・回数・状態封印を満たす全候補を表示する。
- 確定前プレビューはUI側で再計算せず、実際の確定処理と同じ純粋解決器を使用する。
- ダメージは属性・能力補正後の計算値から、肩代わり、障壁、HP適用を順に解決する。AoEは固定対象順で逐次解決する。
- 単体対象技では、ARC到達方式で到達可能でもユニットが存在しない空きマスは合法候補へ含めない。
- 技効果の初期データ表現として、毒・火傷・移動不能・開花封印・回復阻害・一歩強制移動を `SkillDefinition.effects` に追加した。
- 予兆予約技の実行ループ統合は本タスクでは拡張せず、既存の予兆表示契約を維持する。

## 検証

- `pnpm test`
- `pnpm demo:headless`
- `pnpm benchmark:ai`
- `pnpm lint`
- `pnpm build`
- 盤面セル選択とタッチ用対象ボタンが同じ候補IDを確定すること
- 候補増加後も50回の決定論再実行とリプレイ一致を維持すること
