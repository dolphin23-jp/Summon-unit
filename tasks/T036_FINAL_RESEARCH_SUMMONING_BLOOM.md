# T036: 確定研究・召喚・還元・開花

## 目的

T035で`known`となった研究ノードを、研究データ・触媒・必要な個体標本を一括消費して`completed`へ進め、設計図を永久解放する。解放済み設計図から固定レア度倍率で個体を即時召喚し、不要個体の還元、condition修復、解析度しきい値による種単位の開花技解放までをPlayerDataへ原子的に接続する。

## 実装内容

### 確定研究マスター

研究ノードごとに次を定義する。

- 研究データ費
- 固定`catalystId`と必要数
- 固定`speciesId`と必要標本数

R1～R3の確定研究は個体標本を要求しない。R4以上は最低1体の標本を要求する。数量は正の安全な整数とし、ノード、触媒、標本種の参照整合と重複を検証する。

### 確定研究

- `known`状態のノードだけ実行可能
- 条件を満たした研究は100%成功
- 研究データ、触媒、指定した個体標本を一括消費
- locked個体と編成中個体は標本に使用不可
- 標本IDは固定ID順へ正規化し、現在時刻や乱数を使用しない
- ノードを`completed`・開示段階5へ移行
- 対象種の`blueprintUnlocked`を永久にtrueへ変更
- 対象種状態が未作成の場合は、図鑑段階5の種状態を新規作成
- 資源不足、標本不一致、保護個体、状態不一致では部分適用しない

### 召喚

- 設計図解放済み種だけ召喚可能
- ガチャ、失敗、待機なし
- 同種上限、所持上限なし
- 呼び出し側が固定`instanceId`を指定
- 重複instanceIdを拒否
- 新規個体はcondition 10000、未習得、未ロック、未お気に入りで作成

暫定基本費100にレア度倍率を掛ける。

`1, 2, 4, 8, 16, 32, 64, 128, 256, 512`

### 還元

- 召喚費の70%を切り捨てで基本通貨へ返す
- locked個体と編成中個体は還元不可
- 個体削除と通貨増加を一括確定
- 設計図と種状態は失われない

### 修復

- condition不足分を10000へ回復
- 全損時の修復費は召喚費の5%
- 現在のcondition不足割合で按分し、1以上は最低1通貨
- 通貨消費とcondition回復を一括確定
- 戦闘結果からconditionを減らす接続はT038

### 開花研究

種・開花技ごとに次を定義する。

- 解析度しきい値
- 基本通貨費
- 研究データ費

- 設計図解放済み種だけ研究可能
- 解析度がしきい値以上であること
- 開花技が種マスターに属すること
- 解放は種単位で、同種の全個体が利用可能
- 既存のFormationLoadoutは開花技スロットを1つだけ持つ
- 資源消費と`bloomSkillIds`追加を一括確定

## 不変条件

- 確定研究は100%成功し、同一操作で一部の資源だけを減らさない
- 解放済み設計図は還元や個体消費で失われない
- R1～R3は標本不要、R4以上は標本必須
- lockedまたは編成中の個体を消費・還元しない
- 召喚IDに時刻・乱数・表示名を使用しない
- 召喚費、還元額、修復費は整数
- 開花解放は個体ではなく種状態へ保存
- 入力PlayerDataとマスター配列を変更しない

## テスト

`src/progression/t036-progression.test.ts`で以下を検証する。

- R4確定研究の研究データ・触媒・複数種標本消費
- R1確定研究の標本なし完了と種状態新規作成
- locked、編成中、標本不一致、残高不足の原子的拒否
- 設計図永久解放と研究ノード完了
- レア度1～10の固定召喚費
- 即時召喚、重複ID、未解放設計図拒否
- 70%還元と保護個体拒否
- condition比例修復費と全回復
- 解析度しきい値、種単位開花解放、装備1枠
- 100回再実行のJSONバイト一致

検証コマンド:

```bash
pnpm test
pnpm demo:headless
pnpm benchmark:ai
pnpm lint
pnpm build
```
