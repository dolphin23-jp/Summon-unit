# Task 052: ビジュアルテーマ統一・戦闘画面モバイルレイアウト再構成

## 目的

CSSをデザイントークンへ集約し、地域性と視覚階層を持つゲーム画面へ統一する。戦闘画面はモバイルで盤面を維持したまま、操作・タイムライン・ログをタブで切り替える。

## 実装内容

- [x] `src/ui/theme.css`へ全CSSを統合し、色・余白・角丸・影・書体スケールのトークンを定義
- [x] 地域マスターへ`themeColor`を追加し、地域画面と戦闘画面へ反映
- [x] 戦闘画面を盤面＋「操作」「タイムライン」「ログ・AI」の3タブ構成へ再編
- [x] 手動割込・確定前プレビューをタブより前面の固定レイヤーへ配置
- [x] 次回予定を相対順表示へ変更し、生の仮想時刻は詳細時刻モードだけに限定
- [x] 報酬、通貨、ホーム進行、通知へ依存なしのインラインSVGアイコンを導入
- [x] 390px級の画面でページ全体を固定し、盤面と主要操作を画面内に維持

## 不変条件

- 戦闘カーネル、決定論、stable snapshot、セーブスキーマは変更しない
- 全情報と操作へ2タップ以内で到達できる
- 色以外にも文字、記号、形状、aria属性を維持する
- `prefers-reduced-motion`を維持し、新規ランタイム依存を追加しない

## 自動テスト

- [x] 同時刻タイブレークを含む相対順表示
- [x] タブ選択と非表示パネルの状態維持
- [x] CSS単一入口、トークン、前面レイヤー、100dvh構成
- [x] 既存回帰テスト、ベンチマーク、lint、build

## データ・セーブ互換性

セーブスキーマ変更なし。`themeColor`はコンテンツマスターの表示専用フィールドで、contentVersionのみT052へ更新する。
