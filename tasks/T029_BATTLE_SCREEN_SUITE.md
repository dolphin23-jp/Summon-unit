# T029 — 戦闘画面一式

## 目的

T028の対話型戦闘ランナーを、戦況・未来イベント・AI判断を同時に追える戦闘画面へ接続する。

## 実装範囲

- [x] HP・障壁・属性・主要状態最大4件・次回行動・予兆対象を盤面へ常時表示
- [x] 未来12件のタイムラインへユニット行動と予約イベントを混在表示
- [x] 手動候補に触れた際、行動後の次回時刻と位置を仮表示
- [x] 通常一行ログと詳細スコア内訳を切り替える理由パネル
- [x] 表示モデルを純関数として分離し、決定論・上限・予約イベントをテスト

## 仕様上の判断

- 表示は `InteractiveBattleSnapshot` とマスターデータから導出し、React内へ戦闘規則を持ち込まない。
- 状態表示は既知状態を優先し、同条件ではスタック・強度・applicationSequence・IDの固定順で最大4件を選ぶ。
- タイムラインは既存queue順を基礎に、仮イベントを時刻・固定IDで再整列して12件に制限する。
- 仮次回時刻は候補IDから対応する技・移動コストを解決し、現在効果込みの実効Action CostとSpeedから整数tickで算出する。
- 待機候補は連続待機状態を公開していないため、T029では仮次回イベントを出さず、確定後の正式予定のみ表示する。
