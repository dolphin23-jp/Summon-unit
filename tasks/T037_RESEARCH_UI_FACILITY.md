# T037: 研究UI・施設段階

## 目的

T035～T036で実装した研究網、試験研究、確定研究をブラウザUIへ接続し、開示段階に応じた研究ノード表示、候補入力、標本選択、設計図解放を一つの研究画面で操作できるようにする。あわせて研究施設段階を導入し、対象レア度に応じた研究可能範囲をPlayerDataと研究トランザクションへ接続する。

## 実装内容

### 研究施設モデル

施設マスターは段階ごとに次を持つ。

- 連続した整数`stage`（1始まり）
- 研究可能な最大レア度
- 次段階到達時に消費する基本通貨
- 次段階到達時に消費する研究データ

施設段階は昇順で連続し、最大研究レア度は低下しない。段階1は初期状態のため更新費0とする。施設段階が未保存の旧入力は、施設マスターがある場合だけ段階1へ補完する。

### 施設更新

- 現在段階の次に当たる段階だけへ更新可能
- 基本通貨と研究データを原子的に消費
- 最高段階からの追加更新を拒否
- 現実時間待機、失敗抽選、並列キューなし
- 成功時は新しい最大研究可能レア度を返す

### 研究レア度ゲート

T037のサービス境界では、試験研究、確定研究、開花研究の対象種レア度を現在施設段階と照合する。

- 上限以内ならT035～T036の既存純粋トランザクションを実行
- 上限超過なら資源や研究状態を変更せず拒否
- 下位ロジックは施設非依存のまま再利用可能
- 施設状態は各研究結果へ引き継ぐ

### 研究網UI

研究画面は次を表示する。

- 現在施設段階、研究可能レア度、次段階費用
- 開示段階1以上の研究ノード一覧
- ノード状態、開示段階、隣接数
- 選択ノードの詳細
- 研究施設によるロック理由

開示表示:

- 段階0: 画面に表示しない
- 段階1: `?`、兆候説明のみ
- 段階2: 対象種、属性、レア度、隣接情報
- 段階3: 試験研究フォーム
- 段階4: 正確なレシピと確定研究フォーム
- 段階5: 研究完了・設計図解放済み

### 試験研究UI

- 候補として開示済みの研究方式から選択
- 候補素材種・触媒をチェック操作で選択
- 候補の最小・最大要素数を表示
- 消費研究データを明示
- 成功時はレシピ開示、失敗時は方式・素材・触媒の適合数を通知
- 個体・触媒を消費しない

### 確定研究UI

- 研究データ、触媒、標本種・数量を表示
- locked個体と編成中個体を標本候補から除外
- 標本個体を固定instanceIdで選択
- 条件を満たした場合は100%成功
- 資源・個体消費と設計図解放を一括確定

### 垂直スライス構成

T037デモは次の3段階を使用する。

1. 段階1: R2まで
2. 段階2: R4まで
3. 段階3: R5まで

研究網には完了、候補、既知、`?`の各状態を最低1ノードずつ含める。段階1ではR4ノードを表示しつつ操作をロックし、施設更新後に同じ画面で確定研究できることを示す。

## 保存・互換性

T037は`PlayerData.facilities.researchFacilityStage`を追加する。T037デモのschemaVersionは4とする。

既存T032～T036のテスト・デモは施設マスターを持たないため、施設フィールドなしの入力をそのまま扱える。T037画面では施設対応ラッパーがPlayerDataの正規化後に施設状態を再検証・保持する。

施設費用と最大レア度はプレイヤーセーブへ複製せず、`contentVersion`に対応する施設マスターを参照する。保存するのは現在段階だけとする。

## 不変条件

- 施設段階は1から連続する整数
- 最大研究可能レア度は施設更新で低下しない
- 施設更新は一段階ずつ
- 資源不足時に施設段階だけを上げない
- 上限超過研究では研究データ、触媒、個体、研究状態を変更しない
- 開示段階0のノード情報をUIへ表示しない
- `?`ノードで対象種名・レシピを漏らさない
- locked個体・編成中個体を標本候補へ表示しない
- 研究UIに研究成否や経済ルールの正本を置かない
- 現在時刻・乱数・表示名を永続IDまたは順序決定に使用しない

## テスト

`src/progression/t037-research-facility.test.ts`で以下を検証する。

- 旧入力の段階1補完
- 施設更新の通貨・研究データ消費
- 最高段階の拒否
- 段階1でR2試験研究を許可
- 段階1でR4確定研究を原子的に拒否
- 段階2でR4確定研究と開花研究を許可
- 各研究結果で施設段階を維持
- 資源不足更新の原子的拒否
- 100回再実行のJSONバイト一致

検証コマンド:

```bash
pnpm test
pnpm demo:headless
pnpm benchmark:ai
pnpm lint
pnpm build
```

## 手動確認

1. `pnpm dev`でコレクション画面を開く。
2. 「研究網・施設を開く」を押す。
3. 完了、候補、既知、`?`の4状態が異なる表示であることを確認する。
4. Delta候補ノードで方式、素材、触媒を選び、試験研究を行う。
5. 段階1ではR4 Epsilonノードが施設ロックされていることを確認する。
6. 研究施設を段階2へ更新し、基本通貨・研究データの減少とR4解放を確認する。
7. Epsilonノードで標本B・標本Gを選び、確定研究で設計図を解放する。
8. 画面を編成・図鑑へ戻しても施設段階と研究結果が維持されることを確認する。
9. PC幅とスマートフォン幅で操作し、色以外の枠線・模様・テキストでも状態を識別できることを確認する。
