# Task 053: 戦闘演出フレームワーク（フローティングダメージ・モーション・技エフェクト拡張点）

## 目的

戦闘イベントを視覚演出へ変換するプレゼンテーション層を新設し、ダメージ数値ポップ・被弾/回復表現・移動トゥイーン・行動強調を実装する。将来、技ごとの固有エフェクトをデータ登録だけで追加できる拡張点を確定する。

## 背景・参照仕様

- `docs/03_TIMELINE_SYSTEM.md`、`docs/07_BATTLE_EFFECTS.md`、`docs/12_UI_UX.md`（演出3段階）、`docs/16_DATA_SCHEMAS.md`（イベント購読モデル）
- 前提タスク: T051（スプライト盤面）、T052（レイアウト・トークン）
- 戦闘カーネルは変更せず、UIが購読するイベントログを純関数で演出キューへ写像する。

## 実装範囲

- `src/ui/fx/fx-mapper.ts`: `BattleEvent`列を表示専用`FxCommand`列へ決定論的に変換する。
- `src/ui/fx/FxLayer.tsx`: 盤面上へ絶対配置する、入力を遮らないCSSアニメーション層。
- フローティング数値、障壁吸収、軽減/無効、状態変化、被弾フラッシュ、撃破、蘇生、移動、行動リング、予兆点滅を実装する。
- `src/content/skill-fx.ts`: 技IDから斬撃線・投射軌道・範囲波紋を選ぶ表示専用マスター。未登録技は射程・範囲・属性から汎用演出へフォールバックする。
- 標準は全演出、簡略は数値と強調、最小は演出なし。`prefers-reduced-motion`は最小、AUTO×8はコマンド生成をスキップする。

## 対象外

- サウンド、外部パーティクルライブラリ、スプライトシート、カットイン、カメラ演出。

## 守るべき不変条件

- `src/battle/**`、イベントログスキーマ、決定論、stable snapshot、セーブ形式を変更しない。
- 演出再生はUI状態だけで完結し、画面離脱・リセット時に破棄可能とする。
- 演出中も一時停止・手動割込などの入力を遮らない。

## 完了条件

- [x] AUTO×1でダメージ・回復・状態・移動・撃破・予兆を視覚的に追える。
- [x] 演出3段階と`prefers-reduced-motion`が仕様どおり切り替わる。
- [x] `skillFx`へデータを1件追加するだけで対象技のプリセット・色・時間・倍率を変更できる。
- [x] AUTO×8と即時シミュレーションは演出再生を待たず、戦闘結果へ影響しない。

- [x] `skillFx`マスターの未知プリセット、未知技、不正パラメータを検証する。

- [x] 既存の戦闘スナップショット・決定論・ベンチマークを回帰確認する。

## 手動確認

1. 地域Bの火傷・予兆を含む戦闘をAUTO×1で観戦する。
2. 演出「簡略」「最小」とreduced-motion相当で表示量が切り替わることを確認する。
3. 演出中に一時停止、手動一手、再開、リセットを行い、操作が遮られないことを確認する。
