# Task 021: 到達方式・範囲マスク・強制移動

## 目的

技の「選択対象」「到達方式」「着弾後の効果範囲」を分離し、遮蔽・貫通・空きマス指定・全域指定を決定論的に解決する。あわせて、盤外・占有時に失敗する一歩の強制移動を実装する。

## 背景・参照仕様

- `docs/04_BOARD_AND_TARGETING.md`
- `docs/06_SKILL_SYSTEM.md`
- `docs/17_IMPLEMENTATION_ROADMAP.md`
- 前提タスク: T003、T011、T012、T019、T020

## 実装範囲

- 到達方式: CONTACT / DIRECT / PIERCE / ARC / SNIPE / SELF / GLOBAL
- 到達可能マス、実際の着弾点、遮蔽元、貫通対象の解決
- 範囲マスク: 単体 / 横一列 / 縦一列 / 対象+左右 / 対象+後方 / 十字 / 周囲8 / チェイン / 陣営全域
- 押し出し・引き寄せ・左移動・右移動
- 強制移動成功時のユニット位置と盤面占有の同時更新
- 既存SkillDefinitionへの後方互換な既定値
- プレビューや後続の複数対象処理から利用できる純粋API

## 対象外

- 複数対象への実ダメージ・回復・状態付与
- 強制移動失敗時の衝突ダメージ
- 連鎖押し出し、位置交換、複数マス移動
- 強制移動付き技コンテンツの量産
- AI評価への新到達方式・範囲価値の統合
- T022の予兆・設置物・非占有領域

## 守るべき不変条件

- DIRECTは同列の最前敵で停止する
- PIERCEは同列の遮蔽を無視し、通過対象を前から後ろの固定順で返す
- ARCは対象側の空きマスも指定できる
- SNIPEは遮蔽を無視して生存ユニットを指定する
- GLOBALは個別対象・個別マスを選択しない
- 盤面マスクは固定座標順で返す
- チェインは直前の命中地点から最短の未選択対象へ移り、同距離はbattleUnitIdで解決する
- 強制移動は一歩のみで、盤外・陣営境界越え・占有時は失敗する
- 強制移動による交換・連鎖押し出しは行わない
- 同じ入力から同じ解決結果を返し、入力状態を変更しない

## データ・セーブ互換性

- SkillDefinitionの`reachMethod`と`areaMask`は任意項目とする。
- 既存のSINGLE_ENEMY技は`DIRECT`、既存技全般は`SINGLE`を既定値とする。
- `chainTargetCount`はCHAIN範囲のみ指定可能で、未指定時は3体とする。
- 永続セーブは未実装のため既存セーブ互換性への影響はない。

## 仕様上の判断

- `TARGET_AND_BEHIND`の「後方」は、対象自身の陣営におけるローカルrow増加方向とする。
- `VERTICAL_COLUMN`は内部6×3盤面の同列全6マスを対象とする。
- `SURROUNDING_EIGHT`は着弾マス自身を含めず、盤面全体の上下左右・斜め隣接を対象とする。
- チェインは各命中地点から次の最近傍へ移る。距離は内部6×3盤面のマンハッタン距離とする。
- 左右移動は表示向きではなく固定列番号の-1/+1として扱う。
- 強制移動成功は既存の`unit_moved`イベント形式で記録できる。
