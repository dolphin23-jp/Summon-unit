# T039: 地域UI・高速化・進行通知

## 目的

T038で実装したステージマスターと戦闘結果精算をプレイヤーの主要導線へ接続し、地域画面から通常戦、強敵戦、地域ボス、隠し戦闘を選択できるようにする。クリア済みステージには×8表示速度と即時ヘッドレスシミュレーションを提供し、戦闘・研究によって新たに可能になった操作を進行通知として表示する。

## 実装内容

### 地域・ステージ進行マスター

地域マスターは次を持つ。

- 固定`regionId`
- 表示名、概要、表示順
- 解放条件
- 未解放時に地域自体を隠すかどうか

ステージにはT038の戦闘・報酬定義に加え、次のアクセス定義を持たせる。

- 地域内表示順
- 解放条件
- 未解放時にカードを隠すかどうか

解放条件は次の固定4種類とする。

1. 常時解放
2. 指定ステージのクリア
3. 指定した複数ステージの全クリア
4. 指定地域ポイントへの到達

未知の地域・ステージ参照、自己ステージ参照、重複順序、重複条件IDを拒否する。判定はPlayerDataのクリア済みステージと地域ポイントだけを参照し、現在時刻や表示名を使用しない。

### 地域画面

地域画面は次を表示する。

- 基本通貨、研究データ、触媒種数、現在の出撃編成
- 地域一覧、クリア数、地域ポイント、地域解放状態
- ステージ種別、主題、副題、敵数、確定報酬、地域ポイント
- ロック理由
- 地域ポイント到達報酬と受取状態
- 未発見の隠しステージ件数
- 出撃、即時シミュレーション、編成、研究への導線

隠しステージは解放前に名称、主題、敵編成、報酬をDOMへ表示しない。色だけでなく`OPEN / LOCKED / CLEAR`、枠線、破線、模様で状態を区別する。

### ステージ戦闘生成

地域画面からの出撃は次を組み合わせて`HeadlessBattleDefinition`を生成する。

- 選択ステージの敵配置・敵AI
- 選択編成の味方配置・装備・AI
- 共通の種・技マスター
- 共通の行動上限・初回行動コスト

地域またはステージが未解放の場合は戦闘状態を作らず拒否する。UIは解放条件を表示するが、解放可否の正本はprogressionサービスに置く。

### クリア済み高速化

クリア済みステージでは次を解放する。

- 対話型戦闘のAUTO速度×8
- 1戦分の即時シミュレーション

即時シミュレーションは既存の`runHeadlessBattle`をそのまま実行し、結果をT038の`settleStageBattle`へ渡す。戦闘を省略した固定勝利や固定報酬は実装しないため、編成によって敗北・行動上限も発生し、その結果を通常戦闘と同じ規則で精算する。

未クリアステージの即時シミュレーションは拒否する。同じ入力、同じ戦闘ID、同じ追加報酬ロールからはJSONバイト一致する結果を返す。

### 進行通知

PlayerData更新前後を比較し、次を通知する。

- 新地域解放
- 新ステージ解放
- 隠しステージ発見
- 研究ヒント・開示段階更新
- 新設計図解放
- 開花研究可能
- 地域ポイント報酬獲得

通知IDはイベント種別と固定IDから決定的に作成し、同じ通知を重複表示しない。通知から地域、研究、編成・図鑑へ移動できる。通知はT040のセーブ対象にはせず、T039では現在セッションのUI状態として扱う。

### デモ構成

- 地域1「翠風の観測林」

  - 強敵戦: 通常戦クリアで解放

- 地域2「灰燼機構跡」
  - 地域ボスクリアで解放

## 不変条件

- 未解放地域・ステージから戦闘を開始しない
- 隠しステージの情報を解放前にUIへ漏らさない
- 未クリアステージに×8と即時シミュレーションを出さない
- 即時シミュレーション専用の勝敗・報酬ロジックを作らない
- 進行通知の判定を画面コンポーネントへ置かない
- 同じ通知IDを重複表示しない
- 地域ポイント報酬の受取済み状態をUIだけで管理しない
- タッチ・クリックだけで全操作を完結させる

## 手動確認

1. 起動直後に地域画面が表示され、通常戦だけが出撃可能であることを確認する。
2. 通常戦をクリアし、強敵戦解放通知と通常戦の×8・即時シミュレーションを確認する。
3. 通常戦を即時シミュレーションし、地域ポイント40で隠し戦闘が初めて表示されることを確認する。
4. 強敵戦、地域ボスを順にクリアし、次地域が解放されることを確認する。
5. 研究ヒント、設計図、開花可能通知から対応画面へ移動できることを確認する。
6. PC、タブレット、スマートフォン幅で地域一覧、ステージカード、通知操作が利用できることを確認する。
