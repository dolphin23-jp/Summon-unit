# Task 026: 評価スコア一式

## 目的

T025で追加した候補列挙・結果プレビュー・スコア合成基盤へ、通常のオート戦闘で必要になる攻撃・回復・支援・位置・予兆・時間効率の標準評価項目を追加する。

## 背景・参照仕様

- `docs/08_AI_SYSTEM.md`
- `docs/15_BALANCE_GUIDELINES.md`
- `docs/17_IMPLEMENTATION_ROADMAP.md`
- 前提タスク: T025

## 実装範囲

### 攻撃系

- 即時適用ダメージ
- 撃破数と撃破対象最大HPによる撃破価値
- 計算ダメージと実適用ダメージの差によるオーバーキル減点
- 同じ対象へ同等以上の結果を出す安い候補がある場合の重技浪費減点

### 回復・支援系

- 実回復量
- 生回復量と実回復量の差による過剰回復減点
- 対象の次回行動より前に到来する脅威から戦闘不能を防ぐ救援加点
- 障壁・防御等による予測ダメージ軽減
- バフ等が有効利用される予測行動数

### 位置系

- 前方味方から受ける遮蔽
- 後方味方へ提供する遮蔽
- 移動後に攻撃可能な対象数
- 移動後の次行動で見込める最大即時ダメージ
- 固定危険マスの予測ダメージ減点
- 直前のマスへ戻る行動の減点
- 2マス往復を継続する行動の追加減点

### 予兆認識

- 現在の予兆対象マスから外れる行動への回避加点
- 選択後も予兆発動前に再行動できる場合の残留リスク減点
- 再行動できない場合の大きな残留リスク減点
- 再行動できない場合に障壁・防御で軽減する候補への加点

### 時間効率

- 効果補正後の実効行動コスト減点

## 対象外

- 個体作戦・チーム作戦による重み変更（T027）
- 技ごとの積極使用・温存・自動使用禁止ポリシー（T027）
- 一行理由と詳細判断ログ（T027）
- 回帰指標ベンチと固定指標値（T027）
- 現行最小AI・ヘッドレスランナーへの標準評価系の接続
- 回復・障壁・バフ効果そのもののSkillDefinition拡張
- 複数行動先まで読む探索AI
- 敵の全候補行動を枝分かれさせる確率的期待値計算

## 入力境界

現行SkillDefinitionはダメージ倍率を持つが、回復量・障壁容量・バフ持続価値をまだ持たない。そのためT026では戦闘カーネルや技定義を先回りして変更せず、候補ごとに次の支援結果を外部から渡す。

- 生回復量
- 実回復量
- 予測軽減ダメージ
- 有効利用される支援行動数

この支援結果プロジェクションは実行時派生データであり、永続化しない。将来SkillDefinitionに正式な支援効果が追加された際は、効果プレビュー層が同形式を生成する。

危険情報も同様に、固定危険マスと時間付き脅威を明示入力する。時間付き脅威は対象ユニットまたは対象マスと、発動時刻・予測ダメージを持つ。

## 標準重み

数値は仮置きで、`DEFAULT_AI_STANDARD_SCORE_WEIGHTS` に集約する。個別スコア関数へ数値を散在させない。

| 即時ダメージ | 100 / HP |
| 撃破固定加点 | 5000 / 体 |
| 撃破対象最大HP | 10 / HP |
| オーバーキル | -80 / HP |
| 重技浪費 | -20 / 余分なコスト |
| 実回復 | 100 / HP |
| 過剰回復 | -80 / HP |
| 救援 | 6000 / 体 |
| 予測軽減 | 80 / HP |
| 有効支援行動数 | 300 / 行動 |
| 遮蔽を受ける | 400 |
| 味方を遮蔽する | 250 / 体 |
| 攻撃可能対象 | 200 / 体 |
| 次行動予測ダメージ | 25 / HP |
| 固定危険マス | -100 / 予測HP損失 |
| 直前マスへ戻る | -1000 |
| 2マス往復継続 | 追加-1500 |

| 予兆軽減 | 100 / HP |

## 仕様上の判断

- 即時ダメージには実際にHPから減る量のみを使う。
- オーバーキルは計算ダメージから実適用ダメージを引く。
- 重技浪費は、同じ全対象へ同等以上の実ダメージを与える候補の最小コストとの差とする。
- 撃破価値は固定加点に対象最大HPを加え、高耐久対象の撃破をやや高く評価する。
- 救援は、現在HPと既存障壁では次回行動前の脅威に耐えられず、候補の実回復・軽減を加えると生存する場合に成立する。
- 移動後価値は仮想的に行動者だけを移動し、T025の候補列挙・結果プレビューを再利用して計算する。
- 遮蔽は同じ列で、行動者より前に味方がいれば受け、後ろに味方がいれば提供すると評価する。
- 予兆発動前の再行動可否は、候補の実効行動コストと実効速度から整数tickで算出する。
- 同じ入力・重み・支援結果・危険情報・移動履歴から同じ評価列を返す。

## 守るべき不変条件

- 標準スコアラーはReact、DOM、永続化へ依存しない。
- 戦闘状態、技、ユニット、設置物を変更しない。
- 既存の候補列挙・到達・範囲・ダメージ計算を正本として再利用する。
- 全入力値・重み・スコアは安全な整数に限定する。
- 重みは0以上とし、加点・減点の符号はスコアラー側で固定する。
- すべてのスコア項目IDと優先順位は固定する。
