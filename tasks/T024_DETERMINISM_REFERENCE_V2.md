# Task 024: 決定論リファレンスv2

## 目的

Phase 2で追加した効果システムを一つの固定戦闘へ統合し、同じ入力から同じ最終状態・タイムアウト判定・イベント列が得られることを回帰保証する。

## 実装範囲

- `BattleOutcome`への`DRAW`追加
- タイムアウト時の固定比較API
  1. 残存ユニット数
  2. 残存HP割合合計
  3. 与ダメージ合計
  4. 完全同値時のDRAWまたは味方敗北ポリシー
- 残存HP割合を100万分率の整数で比較
- 毒・火傷・障壁・予兆・召喚・領域を含む固定戦闘v2
- 予兆宣言後に対象マスへ召喚し、解決時点の召喚ユニットへ着弾する統合例
- 固定サマリー、固定トレース、固定BattleEventLog
- 100回のバイト一致再実行テスト
- `pnpm demo:headless`へのv2出力統合
- ゴールデン更新規約の文書化

## 対象外

- T025以降の汎用AI評価
- 完成版の対話型戦闘ランナー
- 予兆・召喚専用の完成版表示イベント
- 全ステージコンテンツへの効果配備
- 永続リプレイ・セーブmigration

## 守るべき不変条件

- 比較順は残存数→残存HP割合→与ダメージ→完全同値で固定する
- 残存HP割合は浮動小数点状態を保存せず、整数スケールで比較する
- 環境ダメージや使用者なしのダメージは与ダメージへ加算しない
- 撃破済み・召喚済みユニットを含む固定IDから陣営を解決する
- 同じ初期状態から同じサマリー・トレース・イベント列を返す
- ゴールデンは仕様変更理由を明記したPRでのみ更新する

## データ・セーブ互換性

- `BattleOutcome`へ`DRAW`を追加するが、既存の全滅勝敗判定は変更しない。
- タイムアウト判定は戦闘状態の自然な全滅判定と分離し、終了サマリー・ログ側で使用する。
- 永続セーブは未実装のためmigrationは不要。

## 仕様上の判断

- 残存HP割合は各生存ユニットの`floor(hp × 1,000,000 / maxHp)`を合計する。
- 完全同値の既定値は`DRAW`とし、敗北扱いが必要なモードは`ALLY_DEFEAT`を指定できる。
- v2はAIの賢さではなく効果統合と決定論を固定するため、明示的な時刻と行動順を持つスクリプト戦闘とする。
- ゴールデン更新は`docs/18_GOLDEN_TEST_POLICY.md`に従う。
